👁️ Proofreader

After spending lots of time proofreading Github README's, me and a collegue wanted to automate the process. Proofreader opens a pull request for every push and reviews the README, coming with suggestions and corrections. We used OpenAIs API with function calls to decide when a correction was due, and we integrated the corrections into Githubs GUI.

It does work, but the utility is arguably quite low in it's current state (look at this PR🤪). It is spewing suggestions for every paragraph (or are we just poor writers?). The corrections are not idempotent, so it just keeps on correcting already proofread paragraphs. The project is on hold for now, but could be fixed and extended to incorporate code (e.g, automatic code repairing with pretty pull requests).
